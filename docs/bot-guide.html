Section 1

Before you start

The ClutchGG bot is the Discord-side control room for your tournament. Anything an organizer can do in the website admin portal — set details, import teams, build the bracket, record results — can be done here with a slash command, and the bot keeps everything in sync with clutchgg.in.

The three things you need

  1. Your tournament ID — the code in your tournament's page URL (clutchgg.in/tournament/abc123).
  2. A claim code — a one-time code ClutchGG sends you to link your server (see Step 1 of the workflow).
  3. The bot invited to your Discord server with permission to read channels, send messages, embed links and (optionally) manage roles.
Golden rule. The website is always the source of truth. When you record a result, the bot checks it against the map data on clutchgg.in and refuses a score that doesn't match. If the website is wrong, fix it there (or use /report-issue) — never force a wrong score.

Who can do what

  • ANYONE — info commands (teams, bracket, matches, stats). Safe for players and viewers.
  • ORGANIZER — anything that changes the tournament. You become an organizer when you claim the tournament; add co-organizers with /organizers add.
  • SUPERADMIN — ClutchGG staff onboarding tools.
Section 2 — recommended

Fastest path: the /setup wizard

Instead of running separate commands for each setup step, /setup walks you through everything with popup forms, dropdowns and buttons — no options to remember. It picks up exactly where you left off every time you run it, so you can stop and resume freely.

/setup

Nothing to type as options — running the bare command opens an ephemeral (only-you-see-it) message with a progress bar across the top: ✅ Details · ▶️ Teams · ◻ Bracket · ◻ Seeding · ◻ Done. It figures out which step you're on from the tournament's current state, so re-running /setup later always resumes at the right place — nothing is lost, nothing repeats.

1

Details — one popup form

Shown while the tournament has no start date yet. You get a single button, 📝 Fill in details; clicking it opens a Discord form (modal) with five fields:

FieldRequired?Notes
Start dateYesYYYY-MM-DD, e.g. 2026-08-01. Rejected with an inline error if the format is wrong.
DescriptionNoFree text shown on the tournament's website page.
Max teamsNoA number, 2–128.
Total prize poolNoNumber, in INR.
Per-place prizesNoposition:prize pairs, comma-separated — e.g. 1:25000, 2:15000, 3:10000.

Submitting with a bad date or an out-of-range team count re-shows the same "Try again" button with the specific error listed — your other fields aren't lost. Anything left blank is simply skipped; you can top these up later with /set-details.

2

Teams — import or wait for captains

Shown once details are saved but fewer than 2 teams exist. The wizard doesn't collect teams itself — it points you at /import-teams (or tells you captains can self-register with /register-team, see Section 5) and gives you one button, 🔄 I've imported — continue. Press it any time after teams exist and the wizard re-checks and advances — you can leave the wizard message open and go run the import command in a different message, then come back and click it.

3

Bracket — pick a format from a dropdown

Shown once ≥2 teams exist and no bracket has been generated yet. One dropdown, three options:

OptionBehavior
🗡️ Single eliminationKnockout bracket. Generates round 1 with open slots, then advances to Step 4 (Seeding).
⚔️ Double eliminationWinners + losers bracket. Also advances to Step 4 to seed round 1 of the winners side.
🔁 Round robinEvery team plays every team; pairings and schedule are generated immediately — no seeding step, the wizard jumps straight to Step 5 (Done).

Picking a type fails safely if the tournament has already begun, or if fewer than 2 teams exist by the time you click (someone may have removed a team meanwhile) — you get an inline error instead of a broken bracket.

4

Seeding — one dropdown per open slot

Elimination brackets only (round robin skips this). The embed always names exactly one open slot — e.g. "Now placing: Match 1 · Slot 1 (top)" — with a live count like (0/8 slots filled), plus the full pairing list so far so you can see the whole bracket taking shape. Below it is a single dropdown listing only the teams not yet placed (paginated with ◀ Prev / Next ▶ buttons past 25 teams).

Pick a team and the wizard fills that slot, then automatically re-renders showing the next open slot — no need to re-run the command or pick match/slot numbers yourself. Repeat until every slot is filled (or press Finish later to pause — resuming with /setup picks up on the same open slot).

If your team count doesn't divide evenly into the bracket size, leftover slots stay open as byes once every team is placed — the wizard tells you this and moves straight to Done rather than blocking on slots nobody can fill.

5

Done — what to run next

A green summary embed with the full seeded bracket and four suggested next commands: /update-match (or /auto-schedule to do them all at once) to set match times, /create-team-roles so reminders can ping teams, /post what:upcoming matches to announce, and /lock-tournament action:lock to freeze everything once you're ready. /bracket shows the live result any time.

Why it's faster. No need to look up match/slot numbers — the wizard tracks them for you and only ever offers teams that haven't been placed yet. Every interaction recomputes the current step from the tournament's actual data, so the wizard survives restarts, multiple organizers clicking at once, and picking up days later.
Two-stage tournaments (group stage → playoffs) aren't driven by the wizard — use /set-bracket stage:stage 1 / stage:stage 2 manually, same as the website's separate-stage flow (Section 3, Step 4).
Section 3 — do these in order

Manual step-by-step setup

Prefer typing commands directly, need a two-stage bracket, or want more control than the wizard offers? Follow these steps top to bottom — the same order as the website admin portal. Each step builds on the one before it.

1

Link your server (claim the tournament)

ClutchGG DMs you a one-time claim code. In your own server, run /claim-tournament with your tournament ID, the code, and the channel where the bot should post announcements. You become the first organizer.

/claim-tournament tournament_id:abc123 code:Xk9f2Lp8qR announce_channel:#tournament

Optional: add schedule_channel: and results_channel: to split schedule posts and result cards into separate channels.

2

Set the tournament details

Fill in description, dates, prize pool and format with /set-details. Only the options you fill in are changed — you can run it several times to fill things in gradually.

/set-details description:Summer Valorant Cup 2026 start_date:2026-08-01 max_teams:16 event_type:online prize_total:50000 prize_currency:INR prize_places:1:25000, 2:15000, 3:10000

Dates are YYYY-MM-DD. Prize places are position:amount pairs separated by commas.

3

Import your teams

Run /import-teams with no file first — the bot replies with the .xlsx template (same columns as the website: Team Name, Player Name 1–7, Riot ID 1–7, Role 1–7). Fill it in, then run the command again and attach it.

# 1) get the template /import-teams # 2) upload the filled file /import-teams file:teams.xlsx mode:add to existing teams
Photos aren't imported via Excel — upload player/team photos from the website edit section, exactly like the website rule. Use mode:replace only if you want to wipe and re-import all teams.
Alternative: let captains register themselves with /register-team instead of collecting an Excel file — no spreadsheet round-trip. See Section 4.
4

Generate the bracket

Pick the format with /set-bracket. For a simple one-stage event just choose the type. For a two-stage event (group stage → playoffs) set stage: and run it once per stage.

# single-stage single elimination /set-bracket type:single elimination # or a round robin league, 3 points per win /set-bracket type:round robin points_per_win:3 # two-stage: group stage that qualifies 8 teams, then playoffs /set-bracket type:round robin stage:stage 1 qualifiers:8 /set-bracket type:single elimination stage:stage 2

Replacing an existing bracket needs overwrite:true — it deletes that stage's matches, so the bot makes you confirm.

5

Place teams into the bracket

For elimination brackets, seed teams into round-1 slots with /assign-slot. Run it with no options to see the list of open slots and their numbers.

# see open slots /assign-slot # put "Velocity Gaming" into match 1, top slot /assign-slot match:1 slot:1 team:velocity

Team names match partially — velocity finds "Velocity Gaming". Round robin brackets fill automatically, so you can skip this step for them.

6

Schedule the matches

Fastest way — schedule everything at once with /auto-schedule: give it the first match's date/time, the gap between matches, and how many per day. It times every match in bracket order and rolls over to the next day automatically.

/auto-schedule start_date:2026-08-01 start_time:18:00 gap:60 per_day:4

Only fills matches that don't already have a date — safe to re-run. Add overwrite:true to reschedule everything (completed matches are never touched). Fine-tune any single match afterwards with /update-match.

Or set matches one at a time — date, time, series format and stream link:

/update-match match:1 date:2026-08-01 time:18:30 format:best of 3 stream:https://youtube.com/watch?v=…
Times are India Standard Time (IST) by default — the same wall-clock time your players see on the website. Reminders and the "LIVE" post fire based on this. (A ClutchGG admin can change the zone with the BOT_TIMEZONE setting.)
7

Create team roles (optional but recommended)

Run /create-team-roles to make a mentionable Discord role for every team. Then assign each team's players to their role. Now the 15-minute match reminder can ping the two teams playing.

/create-team-roles

Needs the bot to have the "Manage Roles" permission. After it runs, assign members to the new roles in Server Settings → Members.

8

Announce the schedule

Post the upcoming-matches announcement to your channel so everyone sees the schedule with live website links.

/post what:upcoming matches
9

Lock the tournament when setup is done

Once details, teams, bracket and matchups all exist, freeze bot-side changes so nothing is edited by accident during the event.

/lock-tournament action:lock

You can always /lock-tournament action:unlock to make changes again.

Section 4

Captain self-registration

Instead of collecting an Excel file from every team, let captains register themselves — no spreadsheet round-trip, and you just click Approve.

How it works

WhoDoes what
CAPTAINRuns /register-team in the server. A popup form asks for the team name and players (one per line: Name, Riot#tag, role). Submitting it makes them the team's captain.
ORGANIZERGets pinged with an approval card in the announce channel — team name, captain, full roster. Click ✅ Approve or ❌ Reject.
# a captain runs this — no options, it opens a form /register-team
What happens on Approve. The team is added to the tournament (same shape as an Excel import — shows up in /teams, /roster, works with /setup's seeding step). A Discord role named after the team is created automatically and handed straight to the captain, so match reminders can already tag them.

Who is "the captain"?

Whoever runs /register-team — they're in the Discord server by definition, so there's no ambiguity. The bot links their Discord account to the team the moment they submit the form.

If the captain isn't in Discord yet

Two common cases where a team exists but has no linked captain:

  • Team was Excel-imported — /import-teams has no concept of a captain.
  • Captain joins the server later — after registration, before they've joined.

An organizer links them any time with /set-captain — this also creates/hands over the team role:

/set-captain team:velocity user:@PlayerName
Why isn't this automatic on join? Detecting "a new member just joined and claims to be team X's captain" would need Discord's privileged GuildMembers intent and a way to verify the claim — too easy to spoof. An explicit organizer command is the reliable, abuse-proof path.
Section 5

Running the tournament day-to-day

When a match finishes

Enter the map results on the website (or let the Valorant stat pull do it), then record the series result in Discord. The bot validates your score against the website's map data and advances the bracket automatically.

# list matches waiting for a result /finish-match # record match 3 as a 2–1 win for the first team /finish-match match:3 score:2-1
If the bot rejects your score, it's because clutchgg.in shows something different. Re-check the map results on the website — the website wins. Only use no_stats_override:true when a match genuinely has no map data and you just need to record the series winner.

Sharing results and standings

Post anything to your channel on demand with /post:

/post what:standings table /post what:top players /post what:match result match:3

Fixing things mid-event

  • Wrong roster? /update-roster to add/rename/remove teams and players.
  • Reschedule a match? /update-match match:5 time:20:00.
  • Add a highlight clip? /update-match match:5 clip_title:Ace! clip_url:….
  • Wrong stats on the website? /report-issue flags it to ClutchGG staff.
  • Add a co-organizer? /organizers add.
Multiple tournaments in one server? Commands act on the active tournament. Switch which one with /use-tournament.
Section 6

Automatic announcements

Once your server is linked, the bot posts these on its own — no command needed. Every one can be switched on or off. All times are in the tournament timezone (IST by default).

PostWhen it firesWhere
☀️ Morning scheduleOnce a day, from 8am, listing that day's matches with times and stream links.Schedule channel
⏰ Match reminder15 minutes before a scheduled match. Pings the two teams' roles if they exist.Schedule channel
🔴 Live announcementThe moment a match's start time passes (10-minute grace window — never fires on old matches or restarts).Schedule channel
🏆 Result cardThe moment a match finishes on clutchgg.in — score, MVP + stats, and a unique flavour line.Results channel
👋 Unfinished-match nudgeDM to organizers when a match is 3+ hours past its start time with no result recorded yet.Organizer DMs
🤖 Auto-finish OPT-INThe instant clutchgg.in's map data decides a match, the winner is recorded and the bracket advances — no /finish-match needed. Off by default.—
🌙 End-of-day summaryOnce all of a day's matches are complete — with the standings table.Announce channel
Auto-finish uses the same validation as /finish-match. It only ever records a result the website's own map data has already decided — it can't record a score that doesn't match clutchgg.in, because there's nothing to "type" for it to get wrong.

Turning them on and off

Use /notifications. With no options it shows the current on/off state of everything. Toggle one, or all at once:

# see current settings /notifications # turn off just the live announcements /notifications kind:live-match announcements state:off # opt in to auto-finish /notifications kind:auto-finish decided matches state:on # turn everything off (e.g. during a break) /notifications kind:everything (all of the above) state:off

Section 10

Troubleshooting & FAQ

A command isn't showing up when I type "/"

New commands can take a few minutes to appear after the bot updates. Close and reopen Discord to refresh. If it still doesn't appear, a ClutchGG admin may need to re-register commands.

The bot says "Only organizers can use this command"

You're not on the organizer list for the active tournament. Whoever claimed the tournament can add you with /organizers add.

My score was rejected

This is by design — the score didn't match the map data on clutchgg.in. Fix the map results on the website, then run /finish-match again. The website is always the source of truth.

Reminders / LIVE posts fired at the wrong time

Match times are read in India Standard Time by default (the wall-clock time on the website). If your event runs in a different zone, ask a ClutchGG admin to set BOT_TIMEZONE.

Reminders aren't pinging the teams

Run /create-team-roles, then assign each team's players to their role. The reminder can only tag a role once it exists and members are in it.

Nothing is auto-posting

Check /notifications — a type may be switched off. Also confirm the bot can send messages and embed links in the configured channels.

I lost my claim code

Contact ClutchGG support for a fresh one. After too many wrong attempts the claim command locks you out for a while as a security measure — just wait it out and try again with the correct code.

This server runs more than one tournament

Every command acts on the active tournament. See which are linked and switch between them with /use-tournament.
